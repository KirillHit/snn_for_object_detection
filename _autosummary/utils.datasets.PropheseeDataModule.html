utils.datasets.PropheseeDataModule#

class PropheseeDataModule(data_dir='./data', dataset='gen1', batch_size=4, num_workers=4, num_load_file=8, num_steps=42, time_step=16, time_shift=16, one_label=True)[source]#

Bases: LightningDataModule

Prophesee gen1 and 1mpx datasets

Parameters:

  • data_dir (str, optional) – The directory where datasets are stored. Defaults to “./data”.

  • dataset (str, optional) – The name of the dataset. Supported gen1 and 1mpx.

  • batch_size (int, optional) – Number of elements in a batch. Defaults to 4.

  • num_workers (int, optional) – A positive integer will turn on multi-process data loading with the specified number of loader worker processes. Defaults to 4.

  • num_load_file (int, optional) – Number of concurrently open files in each thread. Defaults to 8.

  • num_steps (int, optional) – Number of frames. Defaults to 16.

  • time_step (int, optional) – Time between frames. Defaults to 16.

  • time_shift (int, optional) – The number of time steps that labels are moved forward relative to their frame. Defaults to 8.

  • one_label (bool, optional) – If true labeling is provided for the last time step only. Intended for training. Defaults to True.

Raises:

ValueError – Invalid dataset name.

setup(stage: str) → None[source]#

Called at the beginning of fit (train + validate), validate, test, or predict. This is a good hook when you need to build models dynamically or adjust something about them. This hook is called on every process when using DDP.

Args:

stage: either 'fit', 'validate', 'test', or 'predict'

Example:

class LitModel(...):
    def __init__(self):
        self.l1 = None

    def prepare_data(self):
        download_data()
        tokenize()

        # don't do this
        self.something = else

    def setup(self, stage):
        data = load_data(...)
        self.l1 = nn.Linear(28, data.num_classes)
test_dataloader()[source]#

An iterable or collection of iterables specifying test samples.

For more information about multiple dataloaders, see this section.

For data processing use the following pattern:

  • download in prepare_data()

  • process and split in setup()

However, the above are only necessary for distributed processing.

Warning

do not assign state in prepare_data
